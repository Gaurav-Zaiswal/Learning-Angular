String interpolation v property binding

Today, I learned about string interpolation and property binding.

  1. String interpolation is used to play around with the variables, as content of two different tags
  2. Every values that are passed through it, is treated as string data type.
    for example
    if you want some content be within <p> dynamically. Then you'd use string interpolation
  1. Property binding is used to dynamically set the predefined properties of any HTML tag
  2. suppose, <p hidden> is a HTML tag, then hidden is the property of it.
  3. property is defined as [property-name]="valiable-name"
  4. List of known properties:
    • diabled
    • hidden
    • required
    • img src
    • background, etc

We can also use String interpolation to set up the property of an html tag, however, since that String
interpolation treates every arguments as string type, even if you try to pass a boolean value such as hidden, disable, it will treat them like strings, and the expected result won't be seen.
